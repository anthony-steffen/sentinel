# GitHub Showcase - Sentinel Fraud Platform

Guia para destacar o projeto no GitHub e aumentar a percepcao de maturidade tecnica.

## 1) Repositorio bem posicionado

Checklist:

1. Nome claro e profissional
2. Descricao objetiva no topo do repo
3. Topics corretos (`fastapi`, `react`, `docker`, `fraud-detection`, `rbac`)
4. README raiz com demo e arquitetura
5. Licenca definida

## 2) Seccoes-chave do README raiz

1. O que e o produto (visao de negocio)
2. Arquitetura e stack
3. Como executar localmente
4. Como testar (credenciais demo)
5. Evidencias de engenharia (RBAC, auditoria, seguranca)
6. Roadmap e proximos passos

## 3) Commits e historico

Boas praticas:

1. Commits pequenos com contexto (`feat`, `fix`, `docs`, `refactor`)
2. Mensagens em ingles, claras e rastreaveis
3. Evitar "update" generico

## 4) Issues e projetos

Mesmo em projeto solo, crie:

1. 3 a 5 issues tecnicas reais
2. Labels (`bug`, `enhancement`, `documentation`, `security`)
3. Milestone "v1 portfolio"

Isso passa maturidade de processo, nao apenas codigo.

## 5) Releases

Crie uma release `v1.0.0-portfolio` com:

1. Escopo entregue
2. Funcionalidades principais
3. Limitacoes conhecidas
4. Proximos passos

## 6) Imagens e prova visual

Adicione no repo:

1. Screenshot desktop
2. Screenshot mobile
3. GIF curto de navegacao

Local recomendado:

- `docs/assets/`

## 7) README de demo para recrutador

Tenha um caminho de teste em 60 segundos:

1. Comando para iniciar demo
2. URL compartilhada por voce
3. Credenciais prontas
4. Escopo esperado por perfil

## 8) Checklist final antes de divulgar

1. `README` revisado
2. Scripts de demo testados
3. Build local funcionando
4. Credenciais demo validadas
5. Mensagem de outreach pronta para recrutador
